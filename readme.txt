Небольшие пояснения по реализации.
1. Файлы HD кеша будут класться в папку temp
2. Конфигурация кеша задается в спринге общим количеством вызовов необходимым для рекеширования
3. Поскольку не было никакой информации ни по размеру ни по типу объектов - был реализован кеш следующим образом:

Сначала мы все объекты кладем в верхний кеш
Когда он переполняется - делаем рекеширование
Рекеширование реализовано по следующей логике:
Сортируем все объекты по частоте вызовов и кладем самые часто используемые в кеш верхнего уровня
Рекеширование оставляет место для одного объекта в кеше верхнего уровня
Когда вообще весь кеш переполняется - добавление объекта вытесняет самый неиспользуемый объект с кеша нижнего уровня

Думал конечно насчет разных методов вытеснения - хотел даже применить адаптивное вытеснение
Но входных данных слишком мало (вернее они слишком общие)
